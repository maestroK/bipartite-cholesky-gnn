## Overview

This repository contains the complete, reproducible implementation accompanying the paper:

> *Bipartite Cholesky Graph Networks for Many-Body Quantum Chemistry*

We show that the Cholesky decomposition of the electron repulsion integral (ERI) tensor naturally induces a **bipartite graph structure** — one node set for molecular orbitals, one for Cholesky auxiliary interaction channels — that can be exploited directly by a graph neural network. The architecture preserves higher-order interaction topology at reduced theoretical complexity compared with scalar-compressed orbital representations.

### Key results (STO-3G basis, six diatomic molecules, 132 geometries)

| Method | Graph Structure | MAE (Ha) ↓ |
|---|---|---|
| **Bipartite-Chol (Ours)** | Factorized Bipartite | **0.0296 ± 0.0176** |
| Uncoupled Orbital Nodes | Orbital Deep-Set | 0.0665 ± 0.0173 |

Zero-shot leave-one-molecule-out MAE ranges from **0.040 Ha** (LiH) to **0.161 Ha** (Li₂), with generalization governed by orbital-structural similarity to the training distribution rather than by nuclear charge asymmetry alone.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/maestroK/bipartite-cholesky-gnn.git
cd bipartite-cholesky-gnn
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv .venv
source .venv/bin/activate      # macOS / Linux
# .venv\Scripts\activate       # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

Or install the `bcgnn` package directly (editable mode for development):

```bash
pip install -e ".[dev]"
```

### Hardware notes

- **Apple Silicon (M1/M2/M3):** The training engine automatically detects MPS and uses it. No configuration needed.
- **NVIDIA GPU:** Install the CUDA-enabled PyTorch wheel before running `pip install -r requirements.txt`. The engine falls back to CUDA if MPS is unavailable.
- **CPU-only:** Everything runs on CPU; expect longer training times (~2–4× slower than MPS on M2).
- **PySCF on Windows:** PySCF does not officially support Windows. Use WSL2 or a Linux/macOS environment.

## Reproducing the paper results

### Quick path: Jupyter notebook (recommended for exploration)

```bash
jupyter notebook notebooks/cholesky_corrected.ipynb
```

Run cells top-to-bottom. The notebook is fully self-contained: it generates the dataset, trains all models, runs every evaluation, and saves all three figures.

---

### Script path (for headless / cluster runs)

#### Step 1 — Generate the dataset

```bash
python scripts/generate_dataset.py --data-dir data --basis sto-3g
```

This runs RHF/ROHF + FCI via PySCF for all 132 geometry/molecule pairs and writes:

- `data/{molecule}_{geom_id}_h.npy` — one-body core Hamiltonian
- `data/{molecule}_{geom_id}_B.npy` — Cholesky vectors
- `data/dataset_index.csv` — metadata (geometry, energies, spin)

Estimated runtime: **5–20 minutes** depending on hardware.

#### Step 2 — Run all experiments

```bash
python scripts/run_experiments.py --data-dir data --results-dir results
```

Runs in sequence:
1. 5-fold cross-validation → `results/cv_predictions.csv`, `results/cv_maes.json`
2. OrbitalOnlyGNN ablation → `results/ablation_maes.json`
3. Leave-one-molecule-out → `results/lomo_results.json`
4. Permutation invariance → `results/invariance.json`
5. Computational scaling benchmark → `results/scaling.json`

To skip specific experiments (e.g., if resuming after a crash):

```bash
python scripts/run_experiments.py --skip-cv --skip-ablation
```

#### Step 3 — Regenerate figures

```bash
python scripts/plot_figures.py --results-dir results --figures-dir figures
```

Produces:
- `figures/fig1_pes_real_predictions.pdf` — PES curves vs. out-of-fold GNN predictions
- `figures/fig2_computational_scaling.pdf` — log-log forward-pass scaling
- `figures/fig3_lomo_latent_anchoring.pdf` — zero-shot MAE vs. ΔZ

## Method summary

### Cholesky decomposition of the ERI tensor

The four-index electron repulsion integral tensor **G** (dimension N² × N²) is positive semi-definite under the Coulomb metric, guaranteeing a Cholesky factorization:

```
g_pqrs ≈ Σ_L  B^L_pq · B^L_rs
```

where **B** ∈ ℝ^(N_aux × N × N) are the Cholesky vectors (density-fitted, via PySCF `df.DF`). This reduces the effective dimension from O(N⁴) to O(N² · N_aux).

### Bipartite graph construction

- **Orbital nodes V_O**: one node per molecular orbital, features = [h_pp, ‖h_p‖₂]
- **Auxiliary nodes V_A**: one node per active Cholesky vector L
- **Edges**: weighted by B^L_pq — connecting orbital pair (p, q) to auxiliary channel L

### Message passing (two-directional)

```
Orbital → Auxiliary:   m_L[h] = Σ_{p,q} B^L_pq · (x_p[h] ⊙ x_q[h])
Auxiliary → Orbital:   m_p[h] = Σ_{L,q} B^L_pq · (h_L[h] ⊙ x_q[h])
```

Implemented via `torch.einsum` tensor contractions (no explicit O(N⁴) edge matrix allocated).

### Δ-machine learning target

The network predicts **correlation energy** ΔE_corr = E_FCI − E_HF rather than total energy. This removes the O(10²) Ha mean-field variance across species, isolating the O(10⁻¹) Ha many-body signal and stabilising training.

## Corrections relative to original code

The code in this repository fixes six issues identified before arXiv submission:

| # | Bug | Fix |
|---|-----|-----|
| 1 | Figure 1 PES predictions were fabricated (`np.random.normal` noise) | Real out-of-fold CV predictions tracked via `oof_pred_corr[val_idx]` |
| 2 | Dataset had 78 geometries; paper claimed 132 | 22 geometries/molecule × 6 molecules = 132 |
| 3 | O₂ `compute_cholesky_integrals` called with `spin=0` (singlet) while FCI used triplet | `spin` parameter added and propagated consistently |
| 4 | LOMO early stopping evaluated on the held-out test molecule | Val split carved from training molecules only |
| 5 | Permutation invariance test used asymmetric B tensor | Symmetric B: `(B_raw + B_raw.T) / 2` |
| 6 | Ablation monkey-patched `FactorizedBipartiteGNN.forward` | Dedicated `OrbitalOnlyGNN` class with clean separation |
